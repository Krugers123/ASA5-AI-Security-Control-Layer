# Roadmap

## Current Public Layer

The public ASA5 Security Enterprise materials provide:

- Core V2 architecture and epistemic boundaries;
- External Interface v0.3 and an OpenAPI 3.1 contract;
- a golden trace integration fixture;
- ASC shadow-mode decision framing;
- TDB public safety properties;
- partner evaluation and validation-status guidance;
- public-safe dashboard screenshots and evidence examples;
- an operator disagreement and falsification path.

## Next Public Steps

- publish a refreshed English Core V2 screenshot set;
- add stable and negative-control golden traces;
- add pressure-without-drift and successful re-anchor fixtures;
- document a public mock-server workflow;
- publish a concise partner integration checklist;
- freeze a reproducible public-safe evaluation baseline;
- invite independent review and record disagreements.

## Partner-Evaluation Steps

- define a scoped runtime and threat model;
- agree telemetry, privacy, retention, and authority boundaries;
- execute shadow analysis on synthetic or authorized data;
- measure false positives and operator usefulness;
- produce a limitation-aware evaluation report;
- expand only if the scoped evidence supports expansion.

## Private Work Not Exposed

Private development continues separately around detector implementation, evidence fusion, thresholds, counterfactual search, recovery optimization, calibration, internal databases, provider adapters, sensitive traces, and deployment logic.

The public repository remains an architecture, evidence-fixture, and integration surface rather than a source-code disclosure.
