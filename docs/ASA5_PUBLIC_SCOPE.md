# ASA 5 Public Scope

## Purpose

This document defines what the public ASA 5 repository is meant to be.

## Public Scope

The public ASA 5 repository is intended to show:

- product identity
- architecture direction
- public-safe protocol framing
- console direction
- selected non-sensitive diagrams

## Not Public By Default

The public ASA 5 repository is not meant to expose:

- full internal implementation logic
- private endpoint design
- private payload contracts
- private deployment strategy
- sensitive enterprise integration details

## Public Role

This repository should function as:

- a public technical window
- a partner-safe architecture layer
- a security-oriented product framing surface

It should not function as the full private project space.
